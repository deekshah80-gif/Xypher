# Contributing to XYpher

Thank you for considering contributing to the XYpher project! Here are some guidelines to help you get started.

## Branching
- **Create a new branch** for each new feature or bug fix you are working on. Use the following naming convention: `feature/<description>` for features and `bugfix/<description>` for bug fixes. 
- To create a branch, run:
  
  ```bash
  git checkout -b feature/my-new-feature
  ```

## Commits
- **Make descriptive commits**. Use clear and descriptive commit messages that explain the purpose of the change.
- Keep your commits **focused on a single task**. If you resolve multiple issues, consider making separate commits for each.

## Pull Requests
- When you are ready to merge your changes, **open a pull request** against the main branch.
- Ensure that your pull request includes a **description of your changes**, links to any related issues, and any testing notes.
- Please **request a review** from at least one other team member before merging.

## Team Workflow
- Follow the project's **coding standards** and **best practices**. Review existing code for examples of style and structure.
- Keep communication open with the team. **Discuss changes and roadblocks** in the relevant channels or during team meetings.
- Make sure to **sync regularly** with the main branch to avoid merge conflicts.

We appreciate your contributions and support in making XYpher better! Feel free to reach out to the maintainers if you have any questions or need assistance.
